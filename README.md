# FUTURE_CS_01 – Vulnerability Assessment Report

## Objective
The objective of this task was to perform a structured Vulnerability Assessment on a live website and document security findings in a professional report format.

## Tools Used
- Nmap
- OWASP ZAP (Passive Scan)
- Browser Developer Tools
- Canva (Report Design)

## Methodology
- Conducted network scanning using Nmap
- Performed passive vulnerability analysis using OWASP ZAP
- Inspected browser headers and client-side behavior
- Identified potential security misconfigurations
- Classified risks as Low / Medium / High

## Key Findings
- Identified potential exposed services
- Detected security header misconfigurations
- Observed minor information disclosure risks
- Provided risk classification and remediation guidance

## Deliverables
- Professional Vulnerability Assessment Report (PDF)
- Supporting screenshots
- Risk classification and remediation steps

## Skills Demonstrated
- Vulnerability identification
- Risk assessment
- Security reporting
- Business-level communication

## Conclusion
This task demonstrates practical understanding of web security assessment techniques and professional security documentation practices.
